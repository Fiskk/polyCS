Important Announcement:

Due to maintenance on DogNET systems, some users may experience warnings when SSHing into Fang. These users will be required to purge the key their machine presently associates with Fang. If you encounter a warning when attempting to SSH into Fang, this means that you were affected.

If you are on a Linux system, the offending key can be removed with a command such as:
ssh-keygen -f "<PATH_TO_.ssh_DIR>/.ssh/known_hosts" -R fang

NOTE: you will have to determine where the system you are using stores SSH keys and adjust the command accordingly.

If you are unable to resolve the issue on your own, please contact us using the support link at the bottom of the page.
