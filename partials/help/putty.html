Connect with SSH using PuTTY


You can use this tutorial to help you remotely connect to a Unix or Linux machine. However, if your client-machine is Linux-based or Unix-based, it is a near-certainty that it has SSH installed already. Nix users can follow this Wiki if they want to use their native SSH. They should begin on step two.


The steps below to connect to remote machines through PuTTY using the SSH Protocol. By default, most Unix and Linux distributions have their SSH Servers enabled, allowing for remote terminal access. This is not limited to Unix and Linux though, as PuTTY can also be used to connect to any device with an SSH Session, such as Cisco or third-party Windows SSH servers. PuTTY is a free implementation of Telnet and SSH for Windows and Unix platforms, along with an xterm terminal emulator.

PuTTY's official webpage can be found here.



1. Opening PuTTY will launch you directly into the SSH connection page.



2. Enter the name of the remote machine that you wish to connect to. Make sure that the 'SSH' protocol is selected. You can either only put in the hosname, or you can use the format 'username@hostname' to explicitly define a username. Also, if you wish to save the connection, you can now name the session in the 'Saved Sessions' area and click save.



3. If it is your first time accessing the server, then you may be greeted by this error message. This is normal, as the keys used for securing the SSH session haven't been exchanged yet. Click yes to continue the connection.



4. If you didn't enter a username in step 2, then enter it now.



5. You will now pe prompted for your password. When you are typing your password, the line will remain blank, but your password is being recorded.



6. You will now be placed on the terminal of the remote machine.



7. PuTTY contains many customizable options and features. The full selection can be seen on the PuTTY Webpage, or in the PuTTY Manual. These control everything from color, text, and window size to specifics for tunneling through SSH sessions, which is covered in another help doc.
