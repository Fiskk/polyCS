Network File System

The home directories of all users are exported from one machine to all other network machines. This means when you log onto a machine in the lab, you are able to access the same files as you have on fang.

The concept behind NFS is that one remote machine (with lots of disk space) is designated as a file server. That file server exports its file system(s) to other machines (clients). On the clients, an image of the file system will be created (mounted) so that it appears to the average user that the file system is on that machine. How it works on the DogNET is that the /home directory is on the server 'spuds' and exported to all other DogNET machines.

If you want to see information on the file systems on the computer you are logged into, type the command df -h.

THe 'df' command will show you the size of the partitions on the machine that you are on, including those that are mounted from other machines.

If you would like to see how much space you are using on the file servers, run the quota command.
