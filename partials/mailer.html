Important Announcement:

Due to maintenance on DogNET systems, some users may experience warnings when SSHing into Fang. These users will be required to purge the key their machine presently associates with Fang. If you encounter a warning when attempting to SSH into Fang, this means that you were affected.

If you are using Linux or OSx or you are using Bash on Ubuntu on Windows, the offending key can be removed with a command such as:
ssh-keygen -f "/<PATH_TO_.ssh_DIR>/.ssh/known_hosts" -R fang
or you can manually remove the lines associated with Fang from the known_hosts file located in the .ssh subdirectory of your home directory.

NOTE: you will have to determine where the system you are using stores SSH keys and adjust the above command accordingly.
